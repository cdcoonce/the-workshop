# Hook Protection Presets

Stack-specific file protection presets for the Guardrails interview domain. Each preset defines a file pattern that Claude should be prevented from editing, along with a human-readable description for the `AskUserQuestion` multiSelect.

---

## General (always included)

These presets apply to every project regardless of stack.

| Pattern | Description | Recommended |
|---|---|---|
| `.git/` | Git internals — never edit directly | Yes |
| `*.key` | Cryptographic private keys | Yes |
| `*.pem` | PEM-encoded certificates and keys | Yes |
| `.claude/settings.json` | Claude Code configuration — prevent accidental overwrites | Yes |

### MultiSelect Format

```
[x] .git/ — Git internals (never edit directly)
[x] *.key / *.pem — Cryptographic keys and certificates
[x] .claude/settings.json — Claude config (prevent accidental overwrites)
```

## Python

Apply when the detected or stated language is Python.

| Pattern | Description | Recommended |
|---|---|---|
| `.env` | Environment variables — often contains secrets, API keys, database credentials | Yes |
| `uv.lock` | uv lock file — auto-generated by `uv lock`, do not edit manually | Yes |
| `poetry.lock` | Poetry lock file — auto-generated by `poetry lock`, do not edit manually | Yes |
| `migrations/` | Database migration files — require manual review and sequential ordering | Yes |
| `__pycache__/` | Compiled Python bytecode — auto-generated, never edit | Yes |
| `.venv/` | Virtual environment directory — auto-generated, never edit | Yes |
| `alembic/versions/` | Alembic migration versions — require manual review | No |

### MultiSelect Format

```
[x] .env — Environment variables with secrets
[x] uv.lock — Lock file (auto-generated by uv)
[x] poetry.lock — Lock file (auto-generated by Poetry)
[x] migrations/ — Database migrations (manual review required)
[x] __pycache__/ — Compiled bytecode (auto-generated)
[x] .venv/ — Virtual environment (auto-generated)
[ ] alembic/versions/ — Alembic migrations (manual review required)
```

### Detection Hints

- Show `uv.lock` only if `uv.lock` or `pyproject.toml` with `[tool.uv]` exists
- Show `poetry.lock` only if `poetry.lock` or `pyproject.toml` with `[tool.poetry]` exists
- Show `migrations/` only if a `migrations/` directory exists or Django/Alembic is detected
- Always show `.env`, `__pycache__/`, `.venv/`

## JavaScript / TypeScript

Apply when the detected or stated language is JavaScript or TypeScript.

| Pattern | Description | Recommended |
|---|---|---|
| `.env` | Environment variables — often contains secrets, API keys, database credentials | Yes |
| `node_modules/` | npm/yarn/pnpm dependencies — auto-installed, never edit | Yes |
| `package-lock.json` | npm lock file — auto-generated by `npm install`, do not edit manually | Yes |
| `yarn.lock` | Yarn lock file — auto-generated by `yarn install`, do not edit manually | Yes |
| `pnpm-lock.yaml` | pnpm lock file — auto-generated by `pnpm install`, do not edit manually | Yes |
| `dist/` | Build output directory — auto-generated by bundler | Yes |
| `build/` | Build output directory — auto-generated by bundler | Yes |
| `.next/` | Next.js build cache — auto-generated | No |
| `.nuxt/` | Nuxt build cache — auto-generated | No |

### MultiSelect Format

```
[x] .env — Environment variables with secrets
[x] node_modules/ — Dependencies (auto-installed)
[x] package-lock.json — npm lock file (auto-generated)
[x] yarn.lock — Yarn lock file (auto-generated)
[x] pnpm-lock.yaml — pnpm lock file (auto-generated)
[x] dist/ — Build output (auto-generated)
[x] build/ — Build output (auto-generated)
[ ] .next/ — Next.js build cache
[ ] .nuxt/ — Nuxt build cache
```

### Detection Hints

- Show `package-lock.json` only if it exists or npm is the detected manager
- Show `yarn.lock` only if it exists or yarn is the detected manager
- Show `pnpm-lock.yaml` only if it exists or pnpm is the detected manager
- Show `.next/` only if Next.js is detected
- Show `.nuxt/` only if Nuxt is detected
- Always show `.env`, `node_modules/`, `dist/`, `build/`

## Combining Presets

When presenting the multiSelect to the user, merge **General** presets with the relevant **stack-specific** presets into a single list. Order: General items first, then stack-specific items.

If the project uses multiple languages (e.g., Python backend + TypeScript frontend), combine presets from both stacks and deduplicate (`.env` appears once, not twice).

If the language is not covered by a specific preset section above, present only the General presets and add a free-text `AskUserQuestion`: "Are there any additional files or directories Claude should never edit?"
