# dbt Documentation

## dbt docs generate

Builds project documentation from model descriptions, tests, and DAG structure. Scans YAML descriptions, test definitions, the full DAG, and column-level metadata. Outputs HTML and `manifest.json` to `target/`.

```bash
dbt docs generate
```

## dbt docs serve

Starts a local web server to browse generated documentation with an interactive DAG visualization.

```bash
dbt docs serve --port 8080
```

## Accessing docs.getdbt.com as Markdown

The official dbt documentation site supports a `.md` URL convention for AI-friendly access. Append `.md` to any page URL to get Markdown:

```
https://docs.getdbt.com/docs/build/models       # HTML page
https://docs.getdbt.com/docs/build/models.md     # Markdown version
```

This is a feature of docs.getdbt.com, not the local `dbt docs serve` command.

## llms.txt and llms-full.txt

docs.getdbt.com provides index files for AI consumption:

- **`https://docs.getdbt.com/llms.txt`** — concise index of all pages with titles and URLs
- **`https://docs.getdbt.com/llms-full.txt`** — full documentation concatenated into one file

These are hosted on docs.getdbt.com and are not generated by local `dbt docs generate`.

## Writing Good Model Descriptions

Write descriptions in YAML schema files. A good description includes:

- **What the model represents** — the business entity or event
- **Key business logic** — important filters, joins, or transformations
- **Grain** — what one row represents (e.g., one row per order)

```yaml
models:
  - name: fct_orders
    description: >
      Completed customer orders at one row per order. Joins Shopify order data
      with Stripe payment breakdowns. Excludes test and cancelled orders.
      Grain: one row per order_id.
    columns:
      - name: order_id
        description: "Primary key. Unique identifier from Shopify."
      - name: total_amount
        description: "Sum of all payment methods (credit card + bank transfer) in USD."
```
